# Limitations

Reset Oracle is an expert-prior hazard model, not a statistically trained prediction model. Its coefficients and uncertainty are judgmental, versioned priors. A single monitored public account is a sparse source, and historical signal windows do not yet contain enough reviewed forward outcomes for retrospective calibration or an analog success rate. Those values remain unavailable rather than being replaced with fixed pseudo-history.

The 3M–9M reset ledger contains reviewed official announcements. The July 9M denominator covers Codex and ChatGPT Work combined; it is not a Codex-only user count. The 10M milestone is a pledge, not a reached or confirmed result. Demo posts and Demo Mode forecasts remain synthetic and are labeled as such.

X API access, Supabase persistence, OpenAI extraction, and real Resend delivery are exercised only when their server credentials are configured. The public site remains operational in Demo Mode. Forecasts are experimental and are not promises about product behavior or official OpenAI announcements.
